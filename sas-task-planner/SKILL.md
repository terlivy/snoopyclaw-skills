# SAS Task Planner

基于 SAS v1.4.0 六阶段门控流水线的任务全生命周期管理。

## 任务生命周期

```
计划 → 方案设计 → 实施 → 测试验证 → 复盘归档 → 运维(可选)
```

## 阶段一：计划

**产出物**：
- 项目需求说明书
- 功能分解清单
- Token/成本/时间估算
- 风险评估报告
- `maintenance_required` 标志（运维阶段是否需要）

**检查点**：
- [ ] 需求理解是否一致
- [ ] 估算是否合理（Token ≤ 预算20% 自动通过）
- [ ] 风险是否已识别

## 阶段二：方案设计

**产出物**：
- 技术方案文档
- 工作流设计
- 验收标准
- 智能体分配方案（附分配佐证）

**分配佐证模板**：
```
因为[原因]，选择[Agent]执行[任务]
```

**阶段门**：
- 低风险 → 自动进入实施
- 高风险（预算超20%、核心架构变更）→ 需用户审批

## 阶段三：实施与执行

**规范**：
- 按方案执行，子任务可并行
- 记录关键决策到工作记忆
- 记录实际 Token 消耗

## 阶段四：测试与验证

**回退规则**：
- 发现缺陷 → 生成修复方案 → 可回到阶段三或阶段二
- 最多重试 3 轮，超过则暂停请人工介入

## 阶段五：复盘与归档

**产出物**：
- 复盘报告
- 案例卡片（成功/失败）
- 准则优化建议

**案例库模板**：
```
案例名称：{名称}
案例状态：{成功/失败/进行中}
关键挑战：{描述}
经验教训：{总结}
对SAS准则的影响：{建议}
```

## 阶段六：运维（可选）

**触发条件**：计划阶段声明 `maintenance_required: true`

## 项目目录结构

```
F:\SC-Workspace\MyTasks\{项目名}\
├── 计划.md        # 项目计划
├── 任务清单.md    # 任务分解
├── 执行记录.md    # 过程记录
├── 评分报告.md    # Agent评分
└── 总结.md        # 项目总结
```

## 质量标准（四项核心要求）

| 要求 | 达标标准 |
|------|----------|
| **快** | 实际时间 ≤ 计划时间 |
| **准** | 一次通过率 ≥ 90% |
| **问题少** | 返工次数 ≤ 1次/任务 |
| **成本低** | 实际成本 ≤ 预算 80% |

## 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 速度 | 25% | 响应时间、执行效率 |
| 质量 | 25% | 输出正确性、完整性 |
| 问题率 | 25% | 出错次数、返工次数 |
| 成本 | 25% | Token消耗、时间成本 |

**综合评分** = 速度分×0.25 + 质量分×0.25 + (10-问题分)×0.25 + 成本分×0.25
